# QQS Сканер

Android-приложение для сканирования QR-кодов фискальных чеков Узбекистана (Soliq), сохранения чеков локально, построения отчётов и экспорта данных.

**Версия:** 1.1 (versionCode 2)  
**Пакет:** `com.example.uzb_qqs_for_dip`

### Установка APK

1. Скачайте файл `.apk` по ссылке выше.  
2. На устройстве откройте **Настройки → Безопасность** (или меню установщика) и при необходимости разрешите установку из источников, отличных от магазина приложений.  
3. Откройте загруженный файл и следуйте подсказкам установщика.

## Возможности

- **Аккаунт** — регистрация профиля, вход в приложение, раздел «Профиль».
- **Добавить** — сканирование QR чека (в том числе через сервисы Google для распознавания штрихкодов).
- **Чеки** — список сохранённых чеков, работа с локальной базой.
- **Отчёт** — формирование отчёта за выбранный период и настройки отчёта.
- **Экспорт** — выгрузка в **CSV**, **XLSX**, **PDF**, изображение чека там, где это предусмотрено в коде.
- Разбор HTML-страниц Soliq и отображение данных чека (суммы, НДС (QQS), поставщик и т.д.).

## Системные требования

- **Android** 7.0 и выше (minSdk 24).
- Доступ в **Интернет** (для загрузки и разбора данных чека с Soliq).
- Разрешения запрашиваются в ходе работы приложения (сканер, файлы — по сценарию ОС).

## Сборка из исходников

Требуется **JDK 17** и **Android SDK** (путь к SDK задаётся в `local.properties`, этот файл не хранится в репозитории).

```bash
./gradlew assembleRelease
```

Готовый APK по умолчанию: `app/build/outputs/apk/release/app-release.apk`.

Для подписи release в проекте используется локальный keystore (`release.keystore`); файл ключа **не входит** в репозиторий по соображениям безопасности. Для своей сборки создайте свой keystore и укажите его в `app/build.gradle.kts`.
